Fix Messages RLS Policies

⚠️ Important: This migration fixes Row Level Security policies that are preventing messages from being marked as read.

Current Issue: Messages appear as unread after refreshing because the database update operations are being blocked by restrictive RLS policies.
What this does:
  • Removes the restrictive update policy
  • Adds a policy allowing authenticated users to update messages
  • Enables marking messages as read to work properly
  • Fixes the issue where read status doesn't persist

Option 1: Quick Fix

Copy and paste the SQL below, then click "Run"

-- Drop existing restrictive update policy DROP POLICY IF EXISTS "Enable update for message sender" ON messages; -- Create a more permissive update policy that allows authenticated users to update messages -- This is specifically needed for marking messages as read (updating read_by_staff and read_by_customer fields) CREATE POLICY "Enable update for all authenticated users" ON messages FOR UPDATE USING (auth.role() = 'authenticated') WITH CHECK (auth.role() = 'authenticated'); -- Ensure RLS is enabled on the table ALTER TABLE messages ENABLE ROW LEVEL SECURITY; -- Grant necessary permissions to authenticated users GRANT UPDATE ON messages TO authenticated; GRANT SELECT ON messages TO authenticated; GRANT INSERT ON messages TO authenticated;
✅ SQL copied! Now paste it in Supabase SQL Editor and click "Run".
